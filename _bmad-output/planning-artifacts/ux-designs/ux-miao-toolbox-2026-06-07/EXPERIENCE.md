# 阿渺工具箱 — 体验骨架

## Foundation

Web 应用，响应式。React 前端 + Ant Design 6 组件体系。Ant Design 承担 80% 的组件工作；品牌纪律是"尊重默认值，只在品牌层覆盖"。`DESIGN.md` 是视觉身份参考，命名了覆盖面；本 spine 是体验层。双角色系统：普通用户 + 管理员，共享同一个应用，管理员多出"管理"入口。暗色模式与亮色模式均需支持。中文界面。

暗色模式实现：antd 6 ConfigProvider + 自定义 `theme.darkAlgorithm` 扩展，手动将 `DESIGN.md` 中定义的 `-dark` 后缀品牌色 token 注入暗色主题。不使用 antd 默认 `darkAlgorithm` 自动计算，因为品牌主色和强调色的暗色变体是精确设计值，非算法推导。

使用规模：十人以内小团队。本版定位：**框架骨架**——认证流程、导航结构、工具接入模板、管理后台结构。具体业务工具（翻译、文生图、文生语音）的交互细节不在本版范围内。

## Information Architecture

| 页面 | 入口 | 目的 |
|------|------|------|
| 登录 / 注册 | 未认证时强制跳转 | 账密登录 + GitHub OAuth |
| 工具列表 | 登录后首页 / 侧栏"工具" | 展示所有可用工具，选择入口 |
| 工具操作页 | 工具列表卡片点击 | 单个工具的操作界面（v1 通用模板） |
| 管理后台-仪表盘 | 侧栏"管理"（仅管理员） | 系统概览：调用量、异常趋势 |
| 管理后台-工具管理 | 管理侧栏 | 启用/禁用工具，查看调用量 |
| 管理后台-调用日志 | 管理侧栏 | 审计日志查询 |
| 管理后台-用户管理 | 管理侧栏 | 用户列表、禁用/限流、角色管理 |
| 个人设置 | 头像/用户名下拉 | 修改密码、绑定 GitHub、登出 |

侧栏结构：
- 普通用户：工具 → [工具列表]
- 管理员：工具 → [工具列表] / 管理 → [仪表盘 / 工具管理 / 调用日志 / 用户管理]
- 底部：头像 + 用户名 → 下拉菜单（个人设置 / 登出）
- 手机端（< 768px）：顶栏右侧头像图标替代侧栏底部菜单，侧栏抽屉关闭时仍可访问个人设置和登出

→ 组合参考：`mockups/tool-list.html`、`mockups/admin-dashboard.html`、`mockups/login.html`。Spine 在冲突时优先。

## Voice and Tone

文案风格。品牌语调与视觉气质参见 `DESIGN.md`。

| Do | Don't |
|---|---|
| "选择一个工具开始使用" | "开启你的 AI 之旅！🚀" |
| "登录已过期，请重新登录" | "Session expired. Please authenticate." |
| "该用户已被禁用" | "用户状态异常，操作已被系统拦截" |
| 短句，主动语态，中性友好 | 感叹号、emoji、过度正式的官话 |
| 管理后台与用户侧语调一致 | 对管理员用更严肃的语气——同一个产品同一个声音 |

## Component Patterns

行为层。视觉规格参见 `DESIGN.md.Components`（或 antd 默认）。

| 组件 | 使用场景 | 行为规则 |
|------|---------|---------|
| Tool card | 工具列表 | 点击整张卡片进入工具操作页。卡片显示：工具图标、名称、简介。禁用工具不显示（FR-15）。键盘可达（Tab + Enter/Space），焦点环参见 `DESIGN.md.Components.tool-card.focus-visible`。 |
| Tool operation template | 工具操作页 | v1 通用模板：标题区 + 参数表单 + 提交按钮 + 结果展示区。各工具通过 YAML 配置驱动参数表单渲染（FR-13）。手机端：表单标签和输入框垂直堆叠（`labelCol: xs 24 / sm 8`），提交按钮全宽（`block`），结果区 `word-break: break-word`。 |
| Admin stat card | 仪表盘 | 展示单一指标（今日调用量、异常数等）。点击可跳转关联详情页。`role="button"` + `tabindex="0"` + `aria-label="{指标名称}: {数值}"`，Enter/Space 触发导航。 |
| Log table | 调用日志 | antd Table，支持按时间/用户/工具/状态筛选。分页加载，不做无限滚动。默认展示最近 24 小时。手机端分页使用 `simple` 模式。 |
| User action menu | 用户管理 | 每行操作列：启用/禁用（Switch）、设置限流（Popover 表单）、角色切换（Dropdown）。所有操作控件需带 `aria-label` 包含用户名（如 `aria-label="禁用用户 张三"`）。操作后 antd Message 确认。 |
| Sidebar nav | 全局 | antd Menu。折叠时只显示图标，展开时图标+文字。当前页高亮 `{sidebar-nav-item-active}`，含 `aria-current="page"`。 |
| User dropdown | 侧栏底部 + 手机顶栏 | antd Dropdown：个人设置、登出。桌面/平板：头像 + 用户名触发。手机端（< 768px）：顶栏右侧头像图标触发，侧栏抽屉内同样保留。 |

## State Patterns

| 状态 | 页面 | 处理 |
|------|------|------|
| 未登录 | 任意页面 | 重定向到登录页。登录后回到原目标页。 |
| 登录失败 | 登录页 | 账密错误：内联提示"用户名或密码错误"。账号锁定：提示"账号已锁定，请 15 分钟后重试"。 |
| OAuth 失败 | 登录页 | "GitHub 授权失败，请重试"。提供账密登录备选。 |
| 无可用工具 | 工具列表 | antd Empty："暂无可用工具"。管理员看到时追加提示"前往管理后台启用工具"。 |
| 工具加载中 | 工具操作页 | antd Skeleton 匹配表单布局。 |
| 工具执行中 | 工具操作页 | 提交按钮 loading 状态 + 结果区 antd Spin。 |
| 工具执行失败 | 工具操作页 | antd Message (error)："请求失败，请稍后重试"。不暴露后端错误详情（FR-9）。 |
| 请求被限流 | 任意 API | antd Notification (warning)："操作过于频繁，请稍后再试"。 |
| 管理后台-无数据 | 仪表盘 / 日志 | antd Empty："暂无数据"。 |
| 管理后台-禁用用户 | 用户管理 | Switch 确认 Popover："确定禁用该用户？"确认后 antd Message："已禁用"。 |
| 管理后台-禁用工具 | 工具管理 | Switch 确认 Popover："禁用后用户将无法看到此工具，确定？" |
| Token 过期 | 任意页面 | 静默尝试 refresh。refresh 失败则重定向到登录页，携带来源路径。 |
| 冷启动 | 工具列表 | antd Skeleton 卡片占位（桌面 3-4 个，手机 1-2 个），数据到达后替换。 |
| 表单动态错误 | 任意表单 | 提交后出现的错误消息通过 `aria-live="polite"` 或 `role="alert"` 通知屏幕阅读器。 |

## Interaction Primitives

- **侧栏导航**：点击切换页面。当前页高亮。折叠/展开按钮在侧栏底部。
- **工具卡片**：点击进入。悬停上浮（参见 `DESIGN.md.Components.tool-card`）。键盘 Tab 可达，焦点环参见 `DESIGN.md.Components.tool-card.focus-visible`。
- **表单提交**：Enter 或点击提交按钮。提交后按钮 loading，防止重复提交。
- **管理操作**：高风险操作（禁用用户、禁用工具、角色变更）需二次确认（antd Popconfirm 或 Modal.confirm）。
- **筛选/搜索**：管理后台表格上方 antd Form 行内筛选，所有筛选字段必须有 `label` 关联（antd `Form.Item label`）。搜索触发按钮点击，不支持实时搜索。手机端：筛选折叠为 antd Collapse，默认收起，仅展开 1-2 个常用筛选。
- **页面刷新**：管理后台仪表盘手动刷新按钮（FR-17）。无自动刷新。
- **深色/浅色切换**：个人设置或侧栏底部切换。切换即时生效（200-300ms CSS transition 平滑过渡），无页面刷新。实现方式：antd 6 ConfigProvider + 自定义 darkAlgorithm 扩展，注入 `DESIGN.md` 定义的 `-dark` 品牌 token。

**禁止：**无限滚动（统一分页）、页面内实时搜索（搜索走按钮）、拖拽排序（v1）、模态框叠加超过一层。

## Accessibility Floor

行为层。视觉对比度参见 `DESIGN.md`（品牌色已通过 WCAG AA 对比度验证）。

- WCAG 2.2 AA 覆盖响应式 Web 表面。
- 键盘可操作：Tab 顺序匹配阅读顺序。Enter 激活按钮/卡片。Esc 关闭弹出层。
- 所有交互元素有可见焦点环（继承 antd `outline` 样式；品牌组件如 tool-card 有自定义焦点指示，参见 `DESIGN.md`）。
- 工具卡片：`role="button"` + `tabindex="0"` + `aria-label="{工具名称} — {工具简介}"`，Enter/Space 触发导航。焦点环参见 `DESIGN.md.Components.tool-card.focus-visible`。
- 管理统计卡片：`role="button"` + `tabindex="0"` + `aria-label="{指标名称}: {数值}"`，Enter/Space 触发导航。
- 管理后台表格：antd Table 自带键盘导航 + `aria-label`。每行操作控件（Switch、Dropdown）必须有唯一 `aria-label` 包含用户名或标识（如 `aria-label="禁用用户 张三"`）。
- 侧栏激活项：`aria-current="page"` 标识当前页面。
- 表单错误：内联错误提示关联到对应输入（antd Form 内建 `aria-describedby`）。动态出现的表单提交错误通过 `aria-live="polite"` 通知屏幕阅读器。
- 管理后台筛选表单：所有筛选字段必须有 `label` 关联（antd `Form.Item label` 或 `aria-label`）。
- 深色/浅色切换：`prefers-color-scheme` 媒体查询作为默认值，用户可手动覆盖。
- 中文字体：确保所有文字在 Noto Sans SC 下渲染清晰，最小字号不低于 12px。
- CJK 正文行高不低于 1.5（WCAG 1.4.12 Text Spacing）。

## Responsive & Platform

| 断点 | 行为 |
|------|------|
| ≥ 992px（antd lg） | 固定侧栏（240px）。工具列表 3 列网格。管理后台表格全宽。 |
| 768–991px（antd md） | 侧栏折叠为图标（64px）。工具列表 2 列。管理后台表格横向滚动。 |
| < 768px（antd sm） | 侧栏变为抽屉。工具列表 1 列。管理后台表格横向滚动 + 筛选折叠为 Collapse（默认收起）。手机顶栏右侧显示头像图标作为用户菜单入口。 |

### 工具操作页响应式

| 断点 | 行为 |
|------|------|
| ≥ 768px | 表单标签和输入框水平排列（antd `labelCol: span 8`）。提交按钮默认宽度。结果区正常展示。 |
| < 768px | 表单标签和输入框垂直堆叠（antd `labelCol: xs span 24`）。提交按钮全宽（`block`）。结果区 `word-break: break-word`，复制按钮固定在结果区右上角。页面顶部显示返回按钮（← 工具列表），仅手机端可见。 |

### 管理后台手机端补充

- 表格操作列使用 `fixed: 'right'` 固定在右侧，避免横向滚动后找不到操作入口。
- 分页使用 `simple` 模式，减少占用空间。
- 登录页手机端：OAuth 按钮和账密表单垂直堆叠，品牌面板隐藏或缩小为顶部标识。

阿渺工具箱是响应式 Web，不是原生 App。手机上可用但以桌面为主要使用场景。管理后台在手机上可用但体验降级（表格横向滚动），日常管理建议桌面操作。

## Inspiration & Anti-patterns

- **借鉴 Notion：** 温暖友好的视觉气质。干净的卡片布局。侧栏导航。
- **借鉴 Vercel Dashboard：** 管理后台的信息密度和表格组织方式。统计卡片 + 筛选 + 表格的三段式。
- **借鉴 Ant Design Pro：** 管理后台的布局框架——侧栏 + 面包屑 + 内容区。但不采用 Pro 的全套模板，只取布局骨架。
- **拒绝——企业安全产品的冷峻感：** 阿渺不需要"安全盾牌"图标、深色仪表盘、红色告警横幅。安全是底色，不是卖点。
- **拒绝——工具市场的花哨展示：** 不做工具评分、热门排行、推荐位。v1 是十人以内的私有工具箱，不是公开市场。
- **拒绝——实时推送和 WebSocket：** v1 仪表盘手动刷新（FR-17）。实时推送是后续优化，不是 v1 需求。
- **拒绝——拖拽式工作流编排：** v1 不做工具间数据联动（PRD §5）。每个工具独立运行。

## Key Flows

### Flow 1 — 日常使用一个 AI 工具（阿渺，普通用户，工作日下午）

1. 阿渺打开浏览器访问阿渺工具箱。
2. 跳转到登录页。选择 GitHub OAuth，点击"使用 GitHub 登录"。
3. GitHub 授权完成，自动跳回工具列表页。页面顶部：`display-sm`："你好，阿渺 👋"。下方是工具卡片网格。
4. 她看到"文本翻译"工具卡片（品牌紫背景，白色文字），点击进入。
5. 工具操作页加载：标题"文本翻译" + 参数表单（源文本输入框、目标语言选择） + 提交按钮。
6. 输入内容，点击"翻译"。按钮进入 loading 状态。
7. **高潮：** 结果区直接展示翻译结果。她点击结果右上角的复制按钮，粘贴到别处使用。无需跳转、无需下载。她从侧栏点回"工具"列表，选择另一个工具继续使用。

失败路径：翻译服务超时 → antd Message (error)："请求失败，请稍后重试"。表单内容保留，她可以直接再次提交。

手机路径：阿渺在手机上打开工具箱。侧栏是抽屉，顶栏右侧有头像图标。点击工具卡片（单列）进入操作页，表单垂直排列，提交按钮全宽，结果区复制按钮固定在右上角。顶部有 ← 返回工具列表。

### Flow 2 — 管理员监控异常调用（Wuxiangyi，管理员，收到异常告警后巡检）

1. Wuxiangyi 登录后，侧栏点击"管理" → "仪表盘"。
2. 仪表盘加载：今日总调用量（比平时高 3 倍）、异常请求趋势图（最近 7 天，今天有尖峰）。异常调用量用暖深橙标记（`{colors.accent}`）。
3. 他注意到调用量异常，点击侧栏切换到"调用日志"。
4. 日志页默认展示最近 24 小时。他切换时间范围到"今天"，看到某个用户的翻译调用量异常集中。
5. 点击侧栏切换到"用户管理"，在搜索框输入该用户名，定位到该用户行。
6. **高潮：** 点击"禁用"Switch，弹出确认："确定禁用该用户？禁用后该用户将无法登录和使用任何工具。"确认。antd Message："已禁用"。该用户的所有请求从这一刻起被鉴权中间件拦截（基于数据库状态实时校验，FR-3）。
7. 回到仪表盘点击刷新，确认调用量开始下降。

失败路径：误禁用 → 在用户管理页面重新启用 Switch。操作记录在审计日志中（FR-18）。
